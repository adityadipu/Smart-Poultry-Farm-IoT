3. Methodology

The proposed Smart Poultry Farm monitoring system is designed as a cohesive IoT solution that integrates renewable energy management, real-time environmental control, and secure remote monitoring. The system development was conducted in three primary phases: Hardware Architecture, Firmware Logic, and Mobile Application Development.

3.1. System Architecture

The overall system architecture is designed using a three-tier model comprising the Physical Hardware Layer, the Communication Layer, and the User Interface Layer. As illustrated in Fig. 1, the system operates bi-directionally: sensors collect environmental data which is transmitted to the cloud, while the user can send control commands back to the physical hardware from the mobile application.

  • Physical Layer: Contains the ESP32-S3 microcontroller, sensors, actuators, and the hybrid power management unit.
  • Communication Layer: Utilizes the MQTT protocol over GPRS (via SIM900A) to ensure lightweight and reliable message transport to a HiveMQ broker.
  • Application Layer: An Android application built with Jetpack Compose that serves as the monitoring dashboard and control panel.

3.2. Hardware Implementation and Hybrid Power System

To provide a clear overview of the electronic design, the complete circuit schematic of the proposed system is presented in Fig. 2. This diagram illustrates the interconnection of all sensors, actuators, the ESP32-S3 controller, and the hybrid power management unit that form the core of the hardware architecture.

Fig. 2: Complete circuit diagram of the Smart Poultry Farm IoT system showing all sensor, actuator, microcontroller, power management, and communication interfaces. (Source: ckt.png)

The core processing unit is the ESP32-S3, selected for its dual-core architecture which allows for the parallel execution of sensor data acquisition and network communication tasks.

A critical innovation of this system is the integration of an intelligent Hybrid Power Management System designed to ensure uninterrupted operation in off-grid scenarios. The power unit utilizes a 12V solar panel and a charge controller to maintain a lead-acid battery. An Automatic Transfer Switch (ATS) circuit was designed using a 5V relay module to switch between the solar battery and a backup AC adapter. To validate the hardware integration, a complete physical prototype was assembled incorporating all power modules, relay mechanisms, and sensor circuitry, as shown in Fig. 3.

The logic for this power switching is governed by the algorithm detailed in Fig. 4. The system continuously monitors the battery voltage via a voltage divider circuit. If the voltage drops below a critical threshold (e.g., 12.0V), the system automatically engages the AC backup to prevent system failure. Conversely, when solar charging restores the battery level (e.g., >13.0V), the system reverts to renewable energy, maximizing energy efficiency.

3.3. Sensor and Actuator Integration

The system incorporates several automated mechanisms to manage feeding, watering, and environmental regulation inside the poultry chamber. The automated feeder mechanism, shown in Fig. 5(a), consists of a servo-driven dispensing gate mounted beneath a feed hopper. An ultrasonic sensor positioned directly above the hopper continuously measures the feed height. When the feed level drops below a predefined threshold, the controller actuates the servo to release a calibrated amount of feed into the trough, ensuring consistent and timely nourishment.

Water availability is maintained through a dedicated water-level monitoring subsystem illustrated in Fig. 5(b). A resistive water-level sensor placed inside a transparent water channel detects variations in water height and transmits this data to the microcontroller. When the water level falls below a critical limit, the system automatically activates the water pump to replenish the channel, preventing dehydration and maintaining hygienic drinking conditions.

To provide a complete understanding of the internal layout, Fig. 5(c) presents a top view of the enclosed poultry chamber displaying all integrated sensors and actuators, including the DHT22 temperature–humidity sensor, MQ135 gas sensor, Light Dependent Resistor (LDR), ventilation fans, heater, day/night lamp, ultrasonic feed-level sensor, and the servo-based feeder mechanism. These components collectively enable real-time monitoring and autonomous regulation of the poultry environment.

3.4. Firmware Logic and Control Algorithms

The firmware for the ESP32-S3 was developed using the Arduino framework C++. To ensure system stability and responsiveness, the code utilizes a non-blocking asynchronous architecture. The main control loop, illustrated in Fig. 6, executes sequentially without using blocking delay() functions, allowing the system to maintain an active MQTT connection while simultaneously reading sensors and controlling actuators.

The firmware performs four key functions in every cycle:

  1. Data Acquisition: Reading data from DHT22 (Temperature/Humidity), MQ135 (Air Quality), LDR (Light Intensity), and Ultrasonic sensors (Feed Level).
  2. Actuator Control: Comparing sensor data against user-configurable thresholds stored in Non-Volatile Storage (NVS) to trigger the Fan, Heater, Pump, or Ventilation system.
  3. Scheduled Feeding: Polling the DS3231 Real-Time Clock (RTC) to trigger the servo motor at precise user-defined feeding times.
  4. Telemetry: Constructing a JSON payload containing sensor readings and power status, which is published to the MQTT broker every 30 seconds.

3.5. Mobile Application and Data Security

The user interface was developed natively for Android using Kotlin and the Jetpack Compose UI toolkit. The application follows a reactive programming model where the UI updates automatically in response to state changes received via MQTT.

The application dashboard (Fig. 7) provides a real-time overview of all farm parameters. It features a dynamic header that displays connectivity status, signal strength, and battery percentage. Below this, a grid of live sensor cards updates instantly as new MQTT packets arrive. A dedicated "Power Status" section allows the user to monitor the active power source (Solar vs. AC) and manually override the automatic power switching logic if necessary.

To ensure system security and prevent unauthorized access to farm controls, a secure authentication flow was implemented as shown in Fig. 8. Upon launching the application, the system checks for a valid login session in local storage. If no session exists, the user is restricted to the Login Activity. Once authenticated, the user gains access to the Dashboard, where they can view live telemetry, perform manual overrides, and configure system thresholds. The application also utilizes a local Room Database to cache incoming sensor data, enabling the generation of historical analytics graphs for energy usage and environmental trends.
